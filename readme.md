    <p class="c15 c34"><span>React was built to solve one problem: </span><span class="c18">building large applications with data that changes over time</span><span>. From the React Docs:</span></p>
    <ul class="c26 lst-kix_dqr48ednmsc2-0 start">
        <li class="c8 c5"><span>Simply express how your app should look at any given point in time, and React will automatically manage all UI updates when your underlying data changes.</span></li>
        <li class="c5 c8"><span>When the data changes, React conceptually hits the &quot;refresh&quot; button, and knows to only update the changed parts.</span></li>
        <li class="c5 c39"><span>React is all about building reusable components. In fact, with React the only thing you do is build components.     </ul>
    <p class="c34 c15"><span class="c4">React is also very fast thanks to the Virtual DOM and diffing algorithm under the hood. When the data changes, React calculates the minimum number of DOM manipulations needed, then efficiently re-renders the component. </span></p>
